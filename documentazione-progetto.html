Vista unica dello stato del progetto

Questa pagina fotografa la struttura attuale del sito. Serve per orientarsi prima di cambiare dati, pipeline o pagine: non esegue script, non modifica file e non sostituisce le pipeline esistenti.

Regola pratica

Prima di eliminare qualcosa, verificare sempre se e' usato da una pagina HTML, da uno script di generazione o da una pipeline intermedia.

Mappa dei file JS

Frontend principali

  • js/nav.js: aggiunge link comuni, mobile menu, dropdown e lazy load MyCombo sulle letture. Alimenta quasi tutte le pagine.
  • js/player.js: costruisce ranking giocatori leggendo rows, playerStats e dati rischio cartellini.
  • js/team-stats.js: pagina statistiche squadra; legge data/active/team-stats-data.js e il JSON merged player-stats.
  • js/pronostico-codex.js: logica tabellone, simulazione, carte squadra e viste del pronostico.
  • js/laboratorio-ai.js: carica e visualizza i file quote JSON in laboratorio.
  • js/mycombo-export.js: arricchisce le letture partita con dati MyCombo da data/mycombo/.

Dataset JS attivi

  • data/active/data.js: rose/squadre attive usate da player.html e schedina.html.
  • data/active/stats.js: statistiche aggregate giocatori per ranking.
  • data/active/team-stats-data.js: dati base della pagina statistiche squadra.
  • data/active/card-risk-data.js: dati rischio cartellini per pagina player.
  • data/active/formations.js, penalty-takers.js, team-insights.js, fifa-rankings.js: supporto pronostico/schedina.

Dataset JS legacy o paralleli

  • js/data.js, js/stats.js, js/team-stats-data.js: ancora caricati da pronostico-codex.html; sembrano paralleli ai bundle in data/active/.
  • js/formations.js, js/formations-insights.js, js/penalty-takers.js, js/team-insights.js, js/fifa-rankings.js: usati dal pronostico principale.
  • js/worldcup-fixtures.js, js/worldcup-results.js, js/tournament-scorers.js, js/reading-predictions.js, js/psychological-factors.js: alimentano scenario torneo e letture.

File JS meno centrali

  • js/referees.js: pagina arbitri.
  • js/captains.js, js/card-risk-data.js, js/final-mycombo-data.js, js/reading-surprise.js, js/round-of-16-readings.js: da verificare per uso effettivo nelle pagine attuali.
  • js/tournament-scorers.js: caricato dal pronostico per capocannonieri/torneo.

4

Pipeline esistenti

Sisal / quote

Trasforma dati quote provider in eventi arricchiti, punteggiati e confrontabili.

Comandi
node scripts/quote-engine.js FILE.json, poi football-intelligence-engine, event-scoring-engine, confidence-engine, probability-engine, value-engine, ranking-engine.
Input
data/providers/quote/ e cartelle provider parallele.
Output
data/events/, data/intelligence/, data/scoring/, data/confidence/, data/probability/, data/value/, data/ranking/.
Pagina
Usata indirettamente da laboratorio, MyCombo e letture.

top-events

Estrae gli eventi migliori per una partita e li prepara per pubblicazione.

Comando
node scripts/build-top-events-mvp.js SLUG
Input
data/quote/ e regole interne nello script.
Output
data/mvp/SLUG/top-events.json e, dopo publish, data/top-events/SLUG-top-events.json.
Pagina
Letture e MyCombo, quando il file e' pubblicato.

ranking

Classifica gli eventi value e aggiunge metadati di mercato.

Comando
node scripts/ranking-engine.js FILE-events.json
Input
data/value/.
Output
data/ranking/ e data/mvp/SLUG/ranking-events.json nelle pipeline MVP.
Pagina
MyCombo/letture tramite output pubblicati.

portfolio

Costruisce combinazioni Safe, Balanced e Aggressive e poi le ottimizza.

Comandi
node scripts/portfolio-builder.js SLUG, node scripts/portfolio-optimizer.js SLUG
Input
data/mvp/SLUG/ranking-events.json e modelli rischio/correlazione.
Output
data/mvp/SLUG/portfolios.json, portfolio-optimized.json, poi data/portfolio/.
Pagina
Letture partita e MyCombo export.

MyCombo

Produce JSON consultabili dalle letture e sincronizza gli output pubblici.

Comandi
node scripts/generate-round-of-16-mycombo.js SLUG, node scripts/generate-daily-mycombo.js SLUG, node scripts/publish-mycombo-pipeline.js SLUG
Input
data/quote/, data/mvp/, ranking e portfolio.
Output
data/mycombo/SLUG.json, round-of-16-summary.json, cartelle pubbliche top-events, ranking, portfolio.
Pagina
lettura.html e letture/*.html tramite js/mycombo-export.js.

player-stats

Scraping multi-provider dei dati player-by-player e merge finale con source per campo.

Comando completo
node scripts/player-stats/run-match.js --match MATCH_ID --espn URL --sofascore URL --fotmob URL --whoscored URL --playwright
Comandi singoli
scrape-espn.js, scrape-sofascore.js, scrape-fotmob.js, scrape-whoscored.js, merge-player-stats.js --match MATCH_ID.
Input
URL provider e raw in data/player-stats/raw/PROVIDER/.
Output
data/player-stats/merged/MATCH_ID.json, raw provider, HTML/JSON debug.
Pagina
statistiche-squadre.html tramite js/team-stats.js.

statistiche squadra

Unisce dati squadra statici e player stats merged in una sola pagina riutilizzabile.

Comando
Nessun comando dedicato: rendering frontend.
Input
data/active/team-stats-data.js e data/player-stats/merged/portugal-spain-2026-07-06.json.
Output
Tab statistiche squadra, calciatori, contesto e stime; metriche derivate per 90 minuti e classifiche automatiche.
Pagina
statistiche-squadre.html.

Possibili file o funzioni da eliminare

Questa lista non implica cancellazione immediata. Sono solo elementi che sembrano duplicati, vecchi fallback o non collegati in modo evidente.

  • scripts/stats/: prima pipeline player-stats; ora esiste anche scripts/player-stats/ piu' professionale.
  • data/player-stats/normalized/ e raw root tipo data/player-stats/raw/portugal-spain-2026-07-06.*.json: sembrano output della pipeline precedente.
  • schedina.html: usa ancora una combinazione vecchia di dataset active e js/pronostico-codex.js.
  • tipster.html: pagina separata, non presente nella navigazione principale come voce dedicata.
  • js/data.js, js/stats.js e analoghi in data/active/: duplicazione potenziale tra dataset completi e attivi.
  • data/ranking/*-events.json e *-ranking-events.json: verificare se sono entrambi necessari.
  • js/final-mycombo-data.js, js/reading-surprise.js, js/round-of-16-readings.js, js/captains.js: controllare se sono ancora richiamati da pagine attive.
  • Dati hardcoded in scripts/build-top-events-mvp.js, scripts/generate-round-of-16-mycombo.js e js/pronostico-codex.js: utili ora, ma candidati a configurazione JSON in futuro.
  • data/archive/: da tenere se serve storico; da escludere da flussi frontend attivi.
6

Prossimi passi consigliati

Tenere

  • statistiche-squadre.html come pagina unica riutilizzabile.
  • scripts/player-stats/ come pipeline principale player-by-player.
  • data/player-stats/merged/ come sorgente finale per il frontend.
  • data/active/ per i bundle frontend delle squadre ancora in gioco.
  • data/mycombo/, data/top-events/, data/portfolio/ come output pubblici.

Unificare

  • Dataset duplicati tra js/ e data/active/.
  • Le due pipeline player-stats in scripts/stats/ e scripts/player-stats/.
  • I file ranking con naming diverso.
  • Le regole hardcoded delle pipeline quote/MyCombo in file JSON di configurazione.

Eliminare dopo verifica

  • Fallback raw/normalized della vecchia pipeline se non servono piu'.
  • Pagine non linkate o obsolete, solo dopo controllo manuale.
  • Script JS non importati da nessuna pagina e non usati da pipeline.
  • Output intermedi duplicati, mantenendo prima un archivio o backup.
